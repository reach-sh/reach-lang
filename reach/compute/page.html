This section describes the common features available in all Reach contexts.0

Comments

 
  1. // single-line comment
  2. /* multi-line
  3.  * comment
  4.  */

Comments are text that is ignored by the compiler. Text starting with // up until the end of the line forms a single-line comment. Text enclosed with /* and */ forms a multi-line comment. It is invalid to nest a multi-line comment within a multi-line comment.1

Blocks

 
  1. { return 42; }
  2. { const x = 31;
  3.   return x + 11; }
  4. { if ( x < y ) {
  5.     return "Why";
  6.   } else {
  7.     return "Ecks"; } }

A block is a sequence of statements surrounded by braces, i.e. { and }.2

Statements

This section describes the statements which are allowed in any Reach context.3

Each statement affects the meaning of the subsequent statements, which is called its tail. For example, if {X; Y; Z;} is a block, then X's tail is {Y; Z;} and Y's tail is {Z;}.4

Distinct from tails are continuations which include everything after the statement. For example, in { {X; Y;}; Z;}, X's tail is just Y, but its continuation is {Y;}; Z;.5

Tails are statically apparent from the structure of the program source code, while continuations are influenced by function calls.6

A sequence of statements that does not end in a terminator statement (a statement with no tail), such as a return statement, continue statement, or exit statement is treated as if it ended with return null;.7

The remainder of this section enumerates each kind of statement.8

const and function

An identifier definition is either a value definition or a function definition. Each of these introduces one or more bound identifiers.9 

 
  1. const DELAY = 10;
  2. const [ Good, Bad ] = [ 42, 43 ];
  3. const { x, y } = { x: 1, y: 2 };
  4. const [ x, [ y ] ] = [ 1, [ 2 ] ];
  5. const [ x, { y } ] = [ 1, { y: 2 } ];
  6. const { x: [ a, b ] } = { x: [ 1, 2 ] };

Valid identifiers follow the same rules as JavaScript identifiers: they may consist of Unicode alphanumeric characters, or _ or $, but may not begin with a digit.10

A value definition is written const LHS = RHS;.11

LHS must obey the grammar:12

 
  1. LHS =
  2.   | id
  3.   | "[" LHS-tuple-seq "]"
  4.   | "{" LHS-obj-seq "}"
  5. LHS-tuple-seq =
  6.   |
  7.   | "..." LHS
  8.   | LHS
  9.   | LHS "," LHS-tuple-seq
  10. LHS-obj-seq =
  11.   |
  12.   | "..." LHS
  13.   | LHS-obj-elem
  14.   | LHS-obj-elem "," LHS-obj-seq
  15. LHS-obj-elem =
  16.   | id
  17.   | propertyName ":" LHS
  18. propertyName =
  19.   | id
  20.   | string
  21.   | number
  22.   | "[" expr "]"

RHS must be compatible with the given LHS. That is, if a LHS is an LHS-tuple-seq, then the corresponding RHS must be a tuple with the correct number of elements. If a LHS is an LHS-obj-seq, then the corresponding RHS must be an object with the correct fields.13

Those values are available as their corresponding bound identifiers in the statement's tail.14

 
  1. function randomBool() {
  2.   return (interact.random() % 2) == 0; };

A function definition, written function FUN(LHS_0, ..., LHS_n) BLOCK;, defines FUN as a function which abstracts its function body, the block BLOCK, over the left-hand sides LHS_0 through LHS_n.15

Function parameters may specify default arguments. The expressions used to instantiate these parameters have access to any variables in the scope of which the function was defined. Additionally, these expressions may reference previous arguments of the function definition. Parameters with default arguments must come after all other parameters.16 

 
  1. function f(a, b, c = a + 1, d = b + c) =>
  2.   a + b + c + d;

The last parameter of a function may be a rest parameter, which allows the function to be called with an arbitrary number of arguments. A rest parameter is specified via ...IDENT, where IDENT is bound to a Tuple containing all the remaining arguments.17

All identifiers in Reach programs must be unbound at the position of the program where they are bound, i.e., it is invalid to shadow identifiers with new definitions. For example,18 

 
  1. const x = 3;
  2. const x = 4;

is invalid. This restriction is independent of whether a binding is only known to a single participant. For example,19 

 
  1. Alice.only(() => {
  2.   const x = 3; });
  3. Bob.only(() => {
  4.   const x = 3; });

is invalid.20

The special identifier _ is an exception to this rule. The _ binding is always considered to be unbound. This means means that _ is both an identifier that can never be read, as well as an identifier that may be bound many times. This may be useful for ignoring unwanted values, for example:21 

 
  1. const [_, x, _] = [1, 2, 3];

return

 
  1. return 17;
  2. return 3 + 4;
  3. return f(2, false);
  4. return;

A return statement, written return EXPR;, where EXPR is an expression, evaluates to the same value as EXPR. As a special case, return; is interpreted the same as return null;.22

A return statement returns its value to the surrounding function application.23

A return statement is a terminator statement, so it must have an empty tail. For example,24 

 
  1. { return 1;
  2.   return 2; }

is invalid, because the first return's tail is not empty.25

Furthermore, a return must have an empty continuation (i.e. it must be in tail position.)26

if

 
  1. if ( 1 + 2 < 3 ) {
  2.   return "Yes!";
  3. } else {
  4.   return "No, waaah!"; }

A conditional statement, written if (COND) NOT_FALSE else FALSE, where COND is an expression and NOT_FALSE and FALSE as statements (potentially block statements), selects between the NOT_FALSE statement and FALSE statement based on whether COND evaluates to false.27

Both NOT_FALSE and FALSE have empty tails, i.e. the tail of the conditional statement is not propagated. For example,28

 
  1. if ( x < y ) {
  2.   const z = 3; }
  3. else {
  4.   const z = 4; }
  5. return z;

is erroneous, because the identifier z is not bound outside the conditional statement.29

A conditional statement may only include a consensus transfer in NOT_FALSE or FALSE if it is within a consensus step, because its statements are in the same context as the conditional statement itself.30

If one branch of a conditional contains a return, then both must.31

switch

 
  1. const mi = Maybe(UInt).Some(42);
  2. switch ( mi ) {
  3.  case None: return 8;
  4.  case Some: return mi + 10; }
  5. switch ( mi ) {
  6.  case None: return 8;
  7.  default: return 41; }

A switch statement, written switch (VAR) { CASE ... }, where VAR is a variable bound to a data instance and CASE is either case VARIANT: STMT ..., where VARIANT is a variant, or default: STMT ..., and STMT is a sequence of statements, selects the appropriate sequence of statements based on which variant VAR holds. Within the body of a switch case, VAR has the type of variant; i.e. in a Some case of a Maybe(UInt) switch, the variable is bound to an integer.32

All cases have empty tails, i.e. the tail of the switch statement is not propagated.33

A switch statement may only include a consensus transfer in its cases if it is within a consensus step, because its statements are in the same context as the conditional statement itself.34

It is invalid for a case to appear multiple times, or be missing, or to be superfluous (i.e. for a variant that does not exist in the Data type of VAR).35

If one case of a switch contains a return, then all must.36

Block statements

A block statement is when a block occurs in a statement position, then it establishes a local, separate scope for the definitions of identifiers within that block. In other words, the block is evaluated for effect, but the tail of the statements within the block are isolated from the surrounding tail. For example,37

 
  1. const x = 4;
  2. return x;

evaluates to 4, but38

 
  1. { const x = 4; }
  2. return x;

is erroneous, because the identifier x is not bound outside the block statement.39

Try/Catch & Throw Statements

 
  1. try {
  2.   throw 10;
  3. } catch (v) {
  4.   transfer(v).to(A); }

A try statement, written try BLOCK catch (VAR) BLOCK, allows a block of code to execute with a specified handler should an exception be thrown.40

A throw statement, written throw EXPR, will transfer control flow to the exception handler, binding EXPR to VAR. Any value that is able to exist at runtime may be thrown. For example, Ints and Arrays are valid values to throw, but a function is not. A throw must have an empty tail.41

Expression statements

 
  1. 4;
  2. f(2, true);

An expression, E, in a statement position is equivalent to the block statement { return E; }.42

Expressions

This section describes the expressions which are allowed in any Reach context. There are a large variety of different expressions in Reach programs.43

The remainder of this section enumerates each kind of expression.44

'use strict'

 
  1. 'use strict';

'use strict' enables unused variables checks for all subsequent declarations within the current scope. If a variable is declared, but never used, there will be an error emitted at compile time.45

strict mode will reject some code that is normally valid and limit how dynamic Reach's type system is. For example, normally Reach will permit expressions like the following to be evaluated:46 

 
  1. const foo = (o) =>
  2.   o ? o.b : false;
  4. void foo({ b: true });
  5. void foo(false);

Reach allows o to be either an object with a b field or false because it partially evaluates the program at compile time. So, without 'use strict', Reach will not evaluate o.b when o = false and this code will compile successfully.47

But, in strict mode, Reach will ensure that this program treats o as having a single type and detect an error in the program as follows:48 

 
  1. reachc: error: Invalid field access. Expected object, got: Bool

The correct way to write a program like this in strict mode is to use Maybe. Like this:49

 
  1. const MObj = Maybe(Object({ b : Bool }));
  3. const foo = (mo) =>
  4.   mo.match({
  5.     None: (() => false),
  6.     Some: ((o) => o.b)
  7.   });
  9. void foo(MObj.Some({ b : true }));
  10. void foo(MObj.None());

unstrict

 
  1. assert(unstrict(() => {
  2.   'use strict';
  3.   // the following fails in strict mode due to a type mismatch
  4.   return 1 != true;
  5. }));

unstrict applies a thunk, ignoring any usage of strict mode. This can be useful when dealing with libraries that are written in strict mode.50

Identifier reference

 
  1. X
  2. Y
  3. Z

An identifier, written ID, is an expression that evaluates to the value of the bound identifier.51

The identifier this has a special meaning inside of a local step (i.e. the body of an only or each expression), as well as in a consensus step (i.e. the tail of publish or pay statement and before a commit statement). For details, see and .52

Function application

 
  1. assert( amount <= heap1 )
  2. step( moveA )
  3. digest( coinFlip )
  4. interact.random()
  5. declassify( _coinFlip )

A function application, written EXPR_rator(EXPR_rand_0, ..., EXPR_rand_n), is an expression where EXPR_rator and EXPR_rand_0 through EXPR_rand_n are expressions that evaluate to one value. EXPR_rator must evaluate to an abstraction over n values or a primitive of arity n. A spread expression (...expr) may appear in the list of operands to a function application, in which case the elements of the expr are spliced in place.53

new f(a) is equivalent to f.new(a) and is a convenient short-hand for writing class-oriented programs.54

Types

Reach's types are represented in programs by the following identifiers and constructors:55

  • Null.56
  • Bool, which denotes a boolean.57
  • UInt, which denotes an unsigned integer. UInt.max is the largest value that may be assigned to a UInt.58
  • Bytes(length), which denotes a string of bytes of length at most length. Bytes of different lengths are not compatible; however the shorter bytes may be padded.59
  • Digest, which denotes a digest.60
  • Address, which denotes an account address.61
  • Contract, which denotes the identifying information of a contract.62

Reach has different representations of contracts across connectors. For example, on Algorand a Contract is an Application ID, but on Ethereum it is an Address.63

  • Token, which denotes a non-network token. Networks discusses how Tokens are represented on specific networks.64
  • Fun([Domain_0, ..., Domain_N], Range), which denotes a function type, when Domain_i and Range are types. The domain of a function is negative position. The range of a function is positive position.65
  • Fun(true, Range), which denotes an unconstrained domain function type, when Range is a type. These functions may only appear in participant interact interfaces.66
  • Tuple(Field_0, ..., FieldN), which denotes a tuple. (Refer to Tuples for constructing tuples.)67
  • Object({key_0: Type_0, ..., key_N: Type_N}), which denotes an object. (Refer to Objects for constructing objects.)68
  • Struct([[key_0, Type_0], ..., [key_N, Type_N]]), which denotes a struct. (Refer to Structs for constructing structs.)69
  • Array(Type_0, size), which denotes a statically-sized array. Type_0 must be a type that can exist at runtime (i.e., not a function type.) (Refer to for constructing arrays.)70
  • Data({variant_0: Type_0, ..., variant_N: Type_N}), which denotes a tagged union (or sum type). (Refer to Data for constructing data instances.)71
  • Refine(Type_0, Predicate, ?Message), where Predicate is a unary function returning a boolean, which denotes a refinement type, that is instances of Type_0 that satisfy Predicate. When a refinement type appears in a negative position (such as in an is or in the domain of a Fun of a participant interact interface), it introduces an assert; while when it is in a positive position, it introduces an assume. Message is an optional string to display if the predicate fails verification.72

For example, if f had type73

 
  1. Fun([Refine(UInt, (x => x < 5))], Refine(UInt, (x => x > 10)))

then const z = f(y) is equivalent to74

 
  1. assert(y < 5);
  2. const z = f(y);
  3. assume(z > 10);
  • Refine(Type_0, PreCondition, PostCondition, ?Messages), where Type_0 is a function type, PreCondition is a unary function that accepts a tuple of the domain and returns a boolean, and PostCondition is a binary function that accepts a tuple of the domain and the range and returns a boolean, denotes a function type with a precondition and postcondition. Preconditions are enforced with assert and postconditions are enforced with assume. Messages is an optional two-tuple of Bytes. The first message will be displayed when the precondition fails verification and the second when the postcondition fails verification.75

For example, Refine(Fun([UInt, UInt], UInt), ([x, y] => x < y), (([x, y], z) => x + y < z)) is a function that requires its second argument to be larger than its first and its result to be larger than its input.76

Object and Data are commonly used to implemented algebraic data types in Reach.77

 
  1. typeOf(x) // type
  2. isType(t) // Bool
  3. is(x, t) // t

The typeOf primitive function is the same as typeof: it returns the type of its argument.78

The isType function returns true if its argument is a type. Any expression satisfying isType is compiled away and does not exist at runtime.79

The is function returns its first argument if it satisfies the type specified by the second argument. If it is not, then the program is invalid. For example, is(5, UInt) returns 5, while is(5, Bool) is an invalid program. The value returned by is may not be identical to the input, because in some cases, such as for functions, it will record the applied to type and enforce it on future invocations. These applications are considered negative positions for Refine.80

Literal values

 
  1. 10
  2. 0xdeadbeef
  3. 007
  4. -10
  5. 34.5432
  6. true
  7. false
  8. null
  9. "reality bytes"
  10. 'it just does'

A literal value, written VALUE, is an expression that evaluates to the given value.81

The null literal may be written as null.82

Numeric literals may be written in decimal, hexadecimal, or octal. Numeric literals must obey the bit width of UInt if they are used as UInt values at runtime, but if they only appear at compile-time, then they may be any positive number. Reach provides abstractions for working with Ints and signed FixedPoint numbers. Ints may be defined by applying the unary + and - operators to values of type UInt. Reach provides syntactic sugar for defining signed FixedPoint numbers, in base 10, with decimal syntax.83

Boolean literals may be written as true or false.84

String literals (aka byte strings) may be written between double or single quotes (with no distinction between the different styles) and use the same escaping rules as JavaScript. Since Bytes types are specialized in their length, literals typically need to be padded to be useful.85

Operator expression

An operator is a special identifier, which is either a unary operator, or a binary operator.86 

 
  1. ! a  // not
  2. - a  // minus
  3. + a  // plus
  4. typeof a
  5. void a

A unary expression, written UNAOP EXPR_rhs, where EXPR_rhs is an expression and UNAOP is one of the unary operators: ! - + typeof void. All the unary operators, besides typeof, have a corresponding named version in the standard library.87

It is invalid to use unary operations on the wrong types of values.88

When applied to values of type UInt, unary - and + operators will cast their arguments to type Int. The unary - and + operations are defined for values of type: Int, and FixedPoint.89

void a evaluates to null for all arguments.90

 
  1. a && b
  2. a || b
  3. a + b
  4. a - b
  5. a * b
  6. a / b
  7. a % b
  8. a | b
  9. a & b
  10. a ^ b
  11. a << b
  12. a >> b
  13. a == b
  14. a != b
  15. a === b
  16. a !== b
  17. a > b
  18. a >= b
  19. a <= b
  20. a < b

Bitwise operations are not supported by all consensus networks and greatly decrease the efficiency of verification.91

A binary expression is written EXPR_lhs BINOP EXPR_rhs, where EXPR_lhs and EXPR_rhs are expressions and BINOP is one of the binary operators: && || + - * / % | & ^ << >> == != === !== > >= <= <. Numeric operations, like + and >, only operate on numbers. Since all numbers in Reach are integers, operations like / truncate their result. Boolean operations, like &&, only operate on booleans. It is invalid to use binary operations on the wrong types of values.92 

 
  1. and(a, b)     // &&
  2. or(a, b)      // ||
  3. add(a, b)     // +
  4. sub(a, b)     // -
  5. mul(a, b)     // *
  6. div(a, b)     // /
  7. mod(a, b)     // %
  8. lt(a, b)      // <
  9. le(a, b)      // <=
  10. ge(a, b)      // >=
  11. gt(a, b)      // >
  12. lsh(a, b)     // <<
  13. rsh(a, b)     // >>
  14. band(a, b)    // &
  15. bior(a, b)    // |
  16. bxor(a, b)    // ^
  17. polyEq(a, b)  // ==, ===
  18. polyNeq(a, b) // !=, !==

All binary expression operators have a corresponding named function in the standard library. While && and || may not evaluate their second argument, their corresponding named functions and and or, always do.93 

 
  1. polyEq(a, b)    // eq on all types
  2. boolEq(a, b)    // eq on Bool
  3. typeEq(a, b)    // eq on types
  4. intEq(a, b)     // eq on UInt
  5. digestEq(a, b)  // eq on Digest
  6. addressEq(a, b) // eq on Addresses
  7. fxeq(a, b)      // eq on FixedPoint
  8. ieq(a, b)       // eq on Int

Equality functions, like ==, ===, !=, and !==, operate on all types. However, values with different types are always not equal. Both arguments must be of the same type. Specialized functions exist for equality checking on each supported type.94

If verifyArithmetic is true, then arithmetic operations automatically make a static assertion that their arguments would not overflow the bit width of the enabled consensus networks. If it is false, then the connector will ensure this dynamically.95

xor

 
  1. xor(false, false); // false
  2. xor(false, true);  // true
  3. xor(true, false);  // true
  4. xor(true, true);   // false

xor(Bool, Bool) returns true only when the inputs differ in value.96

Padding

 
  1. Bytes(16).pad('abc');

Bytes are like Arrays in that they are fixed and exactly sized. This means that two Bytes of different lengths are not interchangeable.97

For example, 'You win!' and 'You lose!' cannot both be provided to an interact function, because the second is one character longer. Most of the time this is good, because it is a signal that you should use a Data type instead, so that the formatting and display logic is entirely controlled by the frontend.98

But, sometimes it is necessary and useful to extend one byte string into a larger size. Each Bytes type has a pad field that is bound to a function that extends its argument to the needed size. A byte string extended in this way is called padded, because it is extended with additional NUL bytes.99

Parenthesized expression

 
  1. (a + b) - c

An expression may be parenthesized, as in (EXPR).100

Tuples

 
  1. [ ]
  2. [ 1, 2 + 3, 4 * 5 ]

A tuple literal, written [ EXPR_0, ..., EXPR_n ], is an expression which evaluates to a tuple of n values, where EXPR_0 through EXPR_n are expressions.101

...expr may appear inside tuple expressions, in which case the spreaded expression must evaluate to a tuple or array, which is spliced in place.102

array

 
  1. const x = array(UInt, [1, 2, 3]);

Converts a tuple of homogeneous values of the specific type into an array.103

Element reference

 
  1. arr[3]

A reference, written REF_EXPR[IDX_EXPR], where REF_EXPR is an expression that evaluates to an array, a tuple, or a struct and IDX_EXPR is an expression that evaluates to a natural number which is less than the size of the array, selects the element at the given index of the array. Indices start at zero.104

If REF_EXPR is a tuple, then IDX_EXPR must be a compile-time constant, because tuples do not support dynamic access, because each element may be a different type.105

If REF_EXPR is a mapping and IDX_EXPR evaluates to an address, then this reference evaluates to a value of type Maybe(TYPE), where TYPE is the type of the mapping.106

Array & tuple length: Tuple.length, Array.length, and .length

 
  1. Tuple.length(tup);
  2. tup.length;
  3. Array.length(arr);
  4. arr.length;

Tuple.length Returns the length of the given tuple.107

Array.length Returns the length of the given array.108

Both may be abbreviated as expr.length where expr evaluates to a tuple or an array.109

Array & tuple update: Tuple.set, Array.set, and .set

 
  1. Tuple.set(tup, idx, val);
  2. tup.set(idx, val);
  3. Array.set(arr, idx, val);
  4. arr.set(idx, val);

Tuple.set Returns a new tuple identical to tup, except that index idx is replaced with val. The idx must be a compile-time constant, because tuples do not support dynamic access, because each element may be a different type.110

Array.set Returns a new array identical to arr, except that index idx is replaced with val.111

Both may be abbreviated as expr.set(idx, val) where expr evaluates to a tuple or an array.112

Array group operations

Array is a Foldable container. Along with the methods of Foldable, the following methods may be used with Arrays.128

Array.iota

 
  1. Array.iota(5)

Array.iota(len) returns an array of length len, where each element is the same as its index. For example, Array.iota(4) returns [0, 1, 2, 3]. The given len must evaluate to an integer at compile-time.129

Array.replicate && .replicate

Fixed-Point Numbers

FixedPoint is defined by233

 
  1. export const FixedPoint = Object({ sign: bool, i: Object({ scale: UInt, i: UInt }) });

FixedPoint can be used to represent numbers with a fixed number of digits after the decimal point. They are handy for representing fractional values, especially in base 10. The value of a fixed point number is determined by dividing the underlying integer value, i, by its scale factor, scale. For example, we could represent the value 1.234 with { sign: Pos, i: { scale: 1000, i : 1234 } } or fx(1000)(Pos, 1234). Alternatively, Reach provides syntactic sugar for defining FixedPoint numbers. One can simply write 1.234, which will assume the value is in base 10. A scale factor of 1000 correlates to 3 decimal places of precision. Similarly, a scale factor of 100 would have 2 decimal places of precision.234 

 
  1. const scale = 10;
  2. const i = 56;
  3. fx(scale)(Neg, i); // represents - 5.6

fx(scale)(i) will return a function that can be used to instantiate fixed point numbers with a particular scale factor.235 

 
  1. const i = 4;
  2. fxint(-i); // represents - 4.0

fxint(Int) will cast the Int arg as a FixedPoint number with a scale of 1.236 

 
  1. const x = fx(1000)(Pos, 1234); // x = 1.234
  2. fxrescale(x, 100);    // => 1.23

fxrescale(x, scale) will convert a fixed point number from using one scale to another. This operation can result in loss of precision, as demonstrated in the above example.237 

 
  1. const x = fx(1000)(Pos, 824345); // x = 824.345
  2. const y = 45.67;
  3. fxunify(x, y);    // => [ 1000, 824.345, 45.670 ]

fxunify(x, y) will convert the fixed point numbers to use the same scale. The larger scale of the two arguments will be chosen. The function will return a 3-tuple consisting of the common scale and the newly scaled values.238

fxadd(x, y) adds two fixed point numbers.239

fxsub(x, y) subtracts two fixed point numbers.240

  1. fxdiv(34.56, 1.234, 10)     // => 28
  2. fxdiv(34.56, 1.234, 100000) // => 28.0064

fxmod(x, y) finds the remainder of dividing x by y.243

fxfloor(x) returns the greatest integer not greater than x.244

fxsqrt(x, k) approximates the sqrt of the fixed number, x, using k iterations of the sqrt algorithm.245

An interval may be constructed with its tuple notation or by function:263

 
  1. // Representing [-10, +10)
  2. const i1 = [Closed, -10, +10, Open];
  3. const i2 = interval(Closed, -10, +10, Open);
  4. const i3 = intervalCO(-10, +10);

For convenience, Reach provides a number of functions for constructing intervals:264

interval(IntervalType, Int, Int, IntervalType) constructs an interval where the first and second argument represent the left endpoint and whether it's open or closed; the third and fourth argument represent the right endpoint and whether it's open or closed.265

intervalCC(l, r) constructs a closed interval from two endpoints of type Int.266

intervalCO(l, r) constructs a half-open interval from two endpoints of type Int where the left endpoint is closed and the right endpoint is open.267

intervalOC(l, r) constructs a half-open interval from two endpoints of type Int where the left endpoint is open and the right endpoint is closed.268

intervalOO(l, r) constructs an open interval from two endpoints of type Int.269

Accessors

leftEndpoint(i) will return the Int that represents the left endpoint of an interval.270

rightEndpoint(i) will return the Int that represents the right endpoint of an interval.271

Intervals may be compared with the following functions:272

intervalEq(l, r) tests whether the intervals are equal.273

intervalNe(l, r) tests whether the intervals are not equal.274

intervalLt(l, r) tests whether the left interval is less than the right interval.275

intervalLte(l, r) tests whether the left interval is less than or equal to the right interval.276

intervalGt(l, r) tests whether the left interval is greater than the right interval.277

intervalGte(l, r) tests whether the left interval is greater than or equal to the right interval.278

Arithmetic Operations

intervalAdd(l, r) adds the two intervals.279

intervalMul(l, r) multiplies the two intervals.281
